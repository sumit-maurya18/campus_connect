# Security Policy

Campus Connect takes security seriously. Although the project is under active development and not yet deployed as a public service, responsible disclosure of security issues is encouraged and appreciated.

---

## Supported Versions

Campus Connect does not currently maintain multiple released versions.

Security fixes are applied only to the **main development branch**.

| Version / Branch | Supported |
|------------------|-----------|
| main             | ✅ Yes    |
| historical code  | ❌ No     |

Once the project reaches a stable release phase, versioned security support will be documented here.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it privately using one of the following methods:

- Email: **security@yourdomain.com** (replace with a real address when available)
- Alternatively, contact the repository owner directly via GitHub

### What to include in your report
Please provide as much detail as possible, including:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact (data exposure, privilege escalation, etc.)
- Any proof of concept, if available

### What to expect
- An acknowledgment of receipt within a reasonable timeframe
- An assessment of the report’s validity and severity
- Updates if the issue is confirmed and addressed

If the report is accepted, a fix will be prioritized and applied.  
If the report is declined, a brief explanation will be provided where appropriate.

---

## Scope

This security policy applies to:
- Backend API logic
- Data handling and validation
- Authentication and authorization mechanisms (when implemented)
- Configuration and environment handling

Issues related to third-party services or dependencies should be reported to their respective maintainers.

---

## Responsible Disclosure

Please allow reasonable time for the issue to be investigated and fixed before any public disclosure.

Good-faith security research and responsible disclosure are welcome.
